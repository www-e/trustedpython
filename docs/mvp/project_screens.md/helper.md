
Support screens
These are not separate business modules, but the UI designer still needs to design them because they affect the full product experience. They save a lot of rework later.

Empty state screen — No listings, no chats, no notifications, no history, no requests.

No internet screen — Offline or retry state.

Error state screen — Generic failure, server error, or failed upload.

Success state screen — Listing published, request submitted, proof uploaded, deal closed.

Confirmation modal — Delete listing, cancel request, close deal, ban user, reopen chat.

Image uploader modal — Pick camera or gallery, preview selected files, upload progress.

Full-screen image viewer — Zoom screenshots and proof images.

Chat locked state screen — Read-only closed chat after mediator finalizes the deal.

Account sold state screen — Listing no longer available and linked to sold page.

Banned account screen — Shown when a user is suspended or banned.

Under review state screen — For flagged listings or paused accounts.

Notification popup templates — Deal request, accepted request, new proof, new message, deal closed.

Role-based navigation shells — Bottom navigation for users, dashboard shell for mediators, sidebar layout for admin.
