
Mediator screens
The mediator experience should feel more like an operations workspace than a normal shopping app. No mediator sign-up screen is needed because mediator accounts are created by admin.

Mediator login screen — Login for mediator-created accounts.

Mediator dashboard screen — Summary of new requests, active deals, closed deals, unread messages, and tier usage.

Incoming requests screen — List of pending deal requests waiting for mediator action.

Request details screen — Shows buyer, seller, listing info, and buttons to accept or decline.

Active deals screen — All ongoing deals assigned to the mediator.

Deal workspace screen — Main mediator screen with access to the three linked chats, deal info, proof attachments, and action buttons.

Group chat screen — Shared room for all three participants.

Buyer-mediator private chat screen — Used for buyer proof and communication.

Seller-mediator private chat screen — Used for seller credential delivery and communication.

Proof review screen — View screenshots, payment proof, credential proof, timestamps, and notes.

Deal action screen — Buttons for mark as sold, cancel deal, close chats, or reopen chats if tier allows.

Close deal confirmation screen — Final step before locking chats and moving item to sold history.

Closed deals screen — Read-only list of finalized deals.

Mediator history screen — Past completed operations with filters by game, date, and outcome.

Mediator public profile screen — Badge, tier, success count, response speed, bio, and trust elements shown to users.

Tier details screen — Explains mediator’s current package, badge, account contribution limit, assistant count, reopen-chat feature, and other benefits.

Assistant management screen — Add, view, disable, or manage mediator assistants allowed under the tier.

Mediator notifications screen — New requests, proof uploaded, unread messages, chat reopened, deal closed.

Mediator settings screen — Password, session control, notification settings, assistant access settings, logout.
