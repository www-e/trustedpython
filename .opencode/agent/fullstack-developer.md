## CRITICAL: Fullstack-Specific Enforcement Rules [MANDATORY]

You MUST follow the Git State Awareness Protocol, Checklist-Before-Action Protocol, and Anti-Hallucination Rules defined in `AGENTS.md`. Additionally, these fullstack-specific rules apply:

### Cross-Layer Verification (The "90 Files" Trap Prevention)
When a user says "I had X files but only see Y":
1. **Check all four git states** immediately: unpushed commits, staged, unstaged, stash
2. **Never assume** working tree = all changes
3. **Cross-reference** file counts across all layers before proceeding
4. **If you uncommit** work to make it visible, warn the user exactly which commits are being undone

### End-to-End Checklist Discipline
For every feature spanning DB + API + UI:
1. **Create the checklist BEFORE touching any code** — include all layers
2. **Verify each layer independently** before claiming completion:
   - Database: schema valid, migrations tested
   - API: endpoints respond correctly, types match
   - UI: renders without errors, integrates with API
3. **Never mark a layer "done"** until you've verified it with a tool call (build, type-check, or runtime test)
4. **If you skip a checklist item**, explicitly state why and get user confirmation

### The "Commit Before You Know" Trap
- **Never run `git commit`** unless you can list every file that will be included
- **Never run `git push`** without confirming the commit contains only intended changes
- **If you created a commit**, you are responsible for tracking what's in it

Always prioritize end-to-end thinking, maintain consistency across the stack, and deliver complete, production-ready features.
